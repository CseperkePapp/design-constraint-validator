# Assessment Starter Pack

Copy-ready task and prompt templates for common assessment work.

Use this pack when your project needs structured analysis work before implementation.

Task templates in this pack are an intentional **lighter variant** of the canonical `templates/TEMPLATE-TASK.md`. They keep the planning-critical sections (Summary, Background, Scope, Acceptance Criteria, Documentation & Versioning Impact, AI Context Load Analysis, Verification Notes) and add an assessment-focused **Deliverables** section, but omit the heavier execution-time sections (Implementation Notes, Risks To Control, Open Questions, AI Execution Quick Reference, Post-Completion Updates). This keeps assessment tasks scannable. When an assessment graduates into implementation work, copy its content onto the full canonical `TEMPLATE-TASK.md` to pick up the execution and closeout sections.

## Included Task Templates

- `TEMPLATE-TASK-REPO-REVIEW.md` — broad technical and workflow review
- `TEMPLATE-TASK-ROOT-CAUSE-INVESTIGATION.md` — scoped investigation with evidence and write-set estimate
- `TEMPLATE-TASK-SWOT-ASSESSMENT.md` — SWOT analysis for product, architecture, or process
- `TEMPLATE-TASK-CODE-QA-ASSESSMENT.md` — quality and regression-risk assessment
- `TEMPLATE-TASK-AI-CAPABILITY-ASSESSMENT.md` — assess AI tools, MCP, skills, agents, and workflows

## Included Prompt Templates

- `TEMPLATE-PROMPT-REPO-REVIEW.md`
- `TEMPLATE-PROMPT-ROOT-CAUSE-INVESTIGATION.md`
- `TEMPLATE-PROMPT-SWOT-ASSESSMENT.md`
- `TEMPLATE-PROMPT-CODE-QA-ASSESSMENT.md`
- `TEMPLATE-PROMPT-AI-CAPABILITY-ASSESSMENT.md`

## Suggested Usage

1. Start from the matching task template in `docs/project-management/tasks/`.
2. If your project uses prompts, pair it with the matching prompt template in `docs/project-management/prompts/`.
3. Keep findings evidence-based and explicitly separate assumptions from verified facts.
4. On closeout, sync the task file and task index.
